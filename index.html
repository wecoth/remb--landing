Проблема

Откуда берётся хаос

«Вечером сел домой, открыл Excel — начал плюсовать, минусовать, полтора часа ушло. А чек где-то потерялся.»

Это знакомая картина для любого кто ведёт несколько объектов. Не потому что не умеешь работать — просто инструментов под эту задачу не было.

  • Кассовые разрывы. Деньги заказчика воспринимаются как свои — остаток неизвестен до конца месяца
  • Чеки теряются. В кармане, в телефоне, в разных чатах — нет единого места
  • Фото россыпью. Сотни снимков в галерее без привязки к объекту и дате
  • Клиент звонит. Вопрос «ну как там?» прерывает работу по несколько раз в день
  • Контроль через обзвон. Чтобы знать статус всех объектов — нужно звонить каждому прорабу
  • Рутина вечером. Вся фиксация откладывается на конец дня и превращается в работу
Система

Из чего состоит

4 компонента которые работают вместе как одно целое

01
Рабочий бот прораба
Основной инструмент. Работает прямо в Telegram — никаких дополнительных приложений. Управление через кнопки, можно говорить голосом.
  • Добавить объект — написал адрес или надиктовал, AI разберёт
  • Фото чека → сумма автоматически в таблицу
  • Голосовое «потратил 4500 на смесь» → зафиксировалось
  • Фото с объекта → автоматически в папку Drive
  • Документы и чеки по нужному объекту
  • Журнал задач по каждому объекту
02
Google Таблица
База данных всей системы. Всё что делает прораб в боте — попадает сюда автоматически. Руководитель видит все объекты в одном месте.
  • Все транзакции: расход/приход, дата, категория, объект
  • Остаток по каждому объекту в реальном времени
  • Сроки: рабочих дней прошло / осталось
  • Список прорабов с доступами
  • Журнал задач и авансов
03
Google Drive
Хранилище всех файлов. При создании объекта — три папки создаются автоматически. Прораб ничего не создаёт руками.
  • 📁 Фото — все снимки с объекта по датам
  • 📁 Документы — сметы, проекты, договоры
  • 📁 Чеки — все квитанции и чеки
  • Папки создаются автоматически при добавлении объекта
  • Доступны через дашборд клиента
04
Клиентский портал
Личный кабинет для заказчика внутри Telegram. Активация одним кодом — и клиент видит свой объект в реальном времени. Без приложений.
  • Статус и прогресс-бар по времени
  • Сколько внесено денег, текущий баланс
  • История платежей
  • Свежие фото с объекта (обновляются автоматически)
  • Документы и чеки
  • Кнопка «Я оплатил» и переход в общий чат

Выгоды

Что это даёт

01
30 секунд вместо вечера с таблицей
Прораб фиксирует прямо на объекте — голосом или фото. Никаких записей «потом», никакой вечерней рутины.
02
Кассовый разрыв виден заранее
Остаток по каждому объекту обновляется в реальном времени. Деньги заказчика — не «вроде есть», а конкретная цифра.
03
Ничего не теряется
Чеки, фото, документы — в одном месте с первого дня. Через полгода можно поднять любой чек с любого объекта.
04
Клиент не звонит
У заказчика есть дашборд — он сам видит прогресс, деньги и фото. Вопрос «ну как там?» отпадает сам собой.
05
Доверие через прозрачность
Личный кабинет для клиента — это конкурентное преимущество при продаже. Прозрачность = спокойный клиент = нормальная работа.
06
Контроль без обзвона
Руководитель видит все объекты в одной таблице. Статус, деньги, задачи — без звонков каждому прорабу.
07
Защита от споров
Архив фото с датами и все чеки — всегда под рукой. Если клиент что-то оспаривает, доказательства уже есть.

Аудитория

Для кого подходит

Подходит
  • Ведёшь 2–3 объекта и больше одновременно
  • Есть прорабы которым нужно что-то фиксировать в поле
  • Клиенты регулярно спрашивают «ну как там?»
  • Хочешь видеть реальный остаток по деньгам не раз в неделю, а всегда
  • Чеки и фото хранятся в разных местах или теряются
  • Хочешь выглядеть профессиональнее перед заказчиком
Не подходит
  • Один объект раз в полгода
  • Всё ведёт один человек и хватает блокнота
  • Нет желания менять привычные процессы
